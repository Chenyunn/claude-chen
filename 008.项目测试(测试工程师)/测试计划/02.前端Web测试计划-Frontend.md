## 一、项目概述

### 1.1 技术栈

| 维度 | 实际情况 |
| --- | --- |
| 框架 | **无（原生 JS，无 React/Vue）** |
| 构建 | **无（零依赖、零打包，静态文件直部署）** |
| 语言 | 原生 JavaScript（ES5/ES2017 混用，非 TS） |
| 状态管理 | 无全局库；模块内变量 + localStorage |
| UI 组件库 | 无第三方库；自研 design-system.css（CSS 变量）+ Material Symbols |
| HTTP | 原生 fetch 自封装（`assets/js/api.js`） |
| 路由 | 无 SPA 路由；多页跳转（`window.location.href` / `<a>`） |
| 测试/Lint | **均未配置** |

### 1.2 目录结构

```
frontend/
├── index.html                 # 首页（财务概览 + 快捷记账）
├── assets/
│   ├── css/design-system.css  # 全局设计系统（725 行）
│   └── js/
│       ├── api.js             # HTTP 封装（fetch + token + 401 处理）
│       ├── auth.js            # 认证业务（登录/注册/登出/鉴权守卫）
│       └── toast.js           # Toast 提示组件
└── pages/
    ├── login.html / register.html   # 登录 / 注册
    ├── index.css                     # 首页样式
    ├── records.html / records.css    # 交易明细
    ├── food.html / food.css          # 餐饮明细
    ├── statistics.html / statistics.css  # 统计图表
    └── profile.html / profile.css    # 设置/我的（含分类管理）
```

### 1.3 启动方式

- **无 package.json / scripts**。
- 开发：浏览器直接打开 `index.html`，或用静态服务器（VS Code Live Server、`python -m http.server`）。
- 构建：无需构建。
- **依赖后端**：必须先启动后端（`localhost:8081`），登录/注册/分类管理才可用。
- **API 地址硬编码**：`assets/js/api.js:22` → `http://localhost:8081/api`（**注释写 8080，实际 8081，以 8081 为准**）。

## 二、测试目标与范围

### 2.1 目标

1. 验证**已接 API 的功能**（登录、注册、登出、获取用户、分类 CRUD）端到端可用。
2. 验证**前端表单校验**与边界。
3. 验证**浏览器兼容性**与响应式。
4. 暴露**前端安全风险**（Token 存储、XSS、鉴权绕过、明文传输）。
5. 明确**原型页/未接 API 页**的边界，仅做可用性检查。

### 2.2 范围内页面与「是否接 API」清单（★测试前务必确认）

| 页面 | 模块 | 是否接 API | 测试重点 |
| --- | --- | :---: | --- |
| `login.html` | 登录 | ✅ | 表单校验、登录态、Token 写入 |
| `register.html` | 注册 | ✅ | 表单校验、注册流程 |
| `index.html` | 首页/快捷记账 | ⚠️ 部分 | 概览展示；**快捷记账仅弹 Toast 不提交（原型）** |
| `records.html` | 交易明细 | ❌ 静态 | 仅静态展示可用性 |
| `food.html` | 餐饮明细 | ❌ 静态 | 仅前端筛选交互 |
| `statistics.html` | 统计图表 | ❌ 静态 | 仅图表展示 |
| `profile.html` | 设置/分类管理 | ✅ | 分类 CRUD、用户信息 |

> **真正打通后端的功能**：登录 / 注册 / 登出 / 获取用户 / 分类管理（在 `profile.html`）。

### 2.3 范围外

- 记账流水入库、交易导出（**前端未接 API，"导出报表"按钮无事件**）。
- 头像上传（"修改头像"无逻辑）、真实支付账户绑定。

## 五、前端安全专项（L5）★本轮核心

| 编号 | 专项 | 测试方法 | 预期/已知缺陷 | 优先级 |
| --- | --- | --- | --- | :---: |
| FE-SEC-01 | **Token 明文存 localStorage** | 登录后 DevTools 查看 localStorage `daily_expense_token` | 已知明文存储，XSS 可窃取 → P1（需后端兜底 + 建议 HttpOnly Cookie） | P1 |
| FE-SEC-02 | **前端鉴权可绕过** | 禁用 JS 或直接访问 `index.html`/`profile.html` | 已知仅 JS 守卫，可绕过 → **依赖后端鉴权，登记 P1** | P1 |
| FE-SEC-03 | **存储型 XSS（innerHTML）** | 创建分类时 name/icon 注入 `<img src=x onerror=alert(1)>`，查看 profile 列表是否执行 | **已知 profile.html:325-340 未转义 → P0** | P0 |
| FE-SEC-04 | **密码明文传输** | DevTools Network 查看登录/注册请求体 | 已知 JSON 明文 POST（依赖 HTTPS） → P1 | P1 |
| FE-SEC-05 | **baseURL 硬编码 http** | 检查 api.js:22 | 已知 `http://localhost:8081`，非 https、无环境区分 → P2 | P2 |
| FE-SEC-06 | 401 自动登出 | 篡改 token 触发 401 | 应清 token 跳登录（验证正向） | P1 |
