# 每日记账 - 产品需求与设计规范

## 1. 产品概念与愿景

**一秒记账，清楚花钱** — 一款令人愉悦、无负担的记账体验，像给自己写便条一样简单，而非繁琐的财务记账。界面温暖宜人，采用"甜蜜生活"美学——柔和的粉色、圆润的线条，以及让每次完成记录都像小型庆祝的微动画。

这不是一款充斥着图表的财务应用。它是一款日常伴侣，让记录支出变得轻松甚至愉快。

## 2. 设计语言

### 美学方向
**"柔软甜品风"** — 灵感来自日本文具设计和糯米糍甜点。干净的白色与温暖的粉色，搭配细腻阴影营造柔和层次感。每个表面都给人温暖的触感。

### 动效理念

每个动画服务于以下三个目的之一：
1. **反馈** — 确认用户操作（按钮点击、记录保存）
2. **引导** — 帮助导航（页面切换、Tab 切换）
3. **愉悦** — 让体验更有活力（保存时的庆祝动效）

| 交互场景 | 动画类型 | 时长 | 缓动 |
|---------|---------|------|------|
| 页面切换 | 左右滑动 | 300ms | ease-out |
| 卡片出现 | 淡入 + 上移 8px | 200ms | ease-out |
| 按钮点击 | 缩放至 0.95 | 100ms | ease-in-out |
| 保存成功 | 打勾 + 弹跳 | 400ms | spring |
| 键盘弹出 | 从底部弹出 | 250ms | ease-out |
| Tab 切换 | 颜色渐变 | 150ms | ease-in-out |

## 4. 功能与交互

### P0：核心功能

#### 快速记账（首页）
- **金额输入**：大号居中输入框，页面加载时自动聚焦，调起数字键盘
- **分类选择**：8 个分类的网格，单击选中，选中态为 scale(1.1) + 边框
- **备注输入**：可选文字输入框，placeholder 为"添加备注..."
- **提交按钮**："记一笔"，带存钱罐图标，金额 > 0 时可用
- **成功反馈**：打勾动画，显示 1 秒后重置表单

#### 消费记录列表（记录）
- **筛选 Tab**：全部 / 今日 / 本周 / 本月
- **列表项**：分类图标、名称、金额（负数、红色）、备注、时间
- **滑动操作**：左滑显示删除按钮
- **点击操作**：打开编辑弹窗
- **空状态**：插图 + "还没有记录，开始记账吧~"

#### 分类管理
- 8 个预设分类（V1 不可编辑）
- 每个分类包含 Emoji、名称和背景色

### P1：重要功能

#### 本月消费概览
- **本月总支出**：大号粗体数字
- **对比上月**：百分比对比，带箭头（↑ 绿色 / ↓ 红色）
- **分类 Breakdown**：横向条形图展示前 3 大分类

#### 消费统计（统计）
- **日历热力图**：30 天网格，颜色深浅表示消费高低
- **分类饼图**：环形图展示各分类占比
- **月度趋势**：近 6 个月的折线图（如有数据）

### P2：增值功能

#### 数据导出
- "我的" Tab 中的导出按钮
- 生成 Excel (.xlsx) 文件
- 列：日期、分类、金额、备注

### 交互细节

| 操作 | 响应 |
|------|------|
| 点击金额输入框 | 键盘弹出，输入框粉色边框高亮 |
| 输入数字 | 金额立即显示，无需格式化 |
| 点击分类 | 放大至 1.1，粉色边框出现，其他取消选中 |
| 点击"记一笔" | 按钮缩小，加载状态，打勾动画，表单调至初始 |
| 左滑记录 | 红色删除按钮出现 |
| 点击删除 | Toast 提示"删除成功"，条目淡出 |
| 点击记录 | 编辑弹窗从底部滑出，预填数据 |
| 下拉列表 | 刷新指示器，重新加载数据 |

### 错误处理

| 场景 | 行为 |
|------|------|
| 金额 = 0 时点击提交 | 按钮抖动，提示"请输入金额" |
| 未选中分类 | 分类行短暂高亮 |
| 删除无确认 | 即时删除 + 带"撤销"的 Toast |
| 消费记录列表为空 | 友好的空状态插图 |

## 5. 组件清单

### 金额显示
- **默认**：48px 粗体，深灰色
- **空状态**：占位符"¥0"，弱化灰色
- **输入中**：数字立即显示，右对齐

### 分类药丸按钮
- **默认**：48x48px 圆角方块，Emoji 居中，分类背景色，不透明度 1
- **选中**：scale(1.1)，2px 粉色边框，阴影
- **悬停**：不透明度 0.9（桌面端）

### 主要按钮（记一笔）
- **默认**：粉色渐变 (#FFB6C1 → #FF8C9A)，白色文字，高度 56px，全宽，24px 圆角，阴影
- **悬停**：亮度 1.05
- **按下**：scale(0.95)，阴影减弱
- **禁用**：不透明度 0.5，无阴影，cursor: not-allowed
- **加载中**：文字替换为加载动画

### 消费记录项
- **默认**：白色卡片，16px 圆角，左侧 Emoji，分类名称 + 备注，右侧金额
- **按下**：背景轻微变暗
- **滑出**：左侧可见红色删除按钮

### Tab 栏
- **容器**：白色背景，顶部 20px 圆角，无阴影，固定底部
- **Tab 项**：图标 24px + 文字 10px，垂直堆叠
- **未选中**：灰色图标 + 文字 (#8E8E93)
- **选中**：粉色图标 + 文字 (#FF8C9A)

### 弹窗/底部面板
- **遮罩层**：rgba(0,0,0,0.3)，淡入 200ms
- **内容**：白色，顶部 20px 圆角，上滑 300ms
- **拖动手柄**：40x4px 灰色药丸，居中，距顶部 8px

### Toast 提示
- **成功**：薄荷绿背景，白色文字，打勾图标
- **错误**：珊瑚橙背景，白色文字
- **可撤销**：附加"撤销"按钮
- **时长**：2 秒，上滑淡出

## 6. 技术方案

### 技术栈
- **框架**：原生 HTML/CSS/JavaScript（无需构建步骤）
- **存储**：localStorage 持久化数据
- **图表**：简单 SVG 图表（无外部依赖）
- **导出**：SheetJS (xlsx) 通过 CDN 引入

### 数据模型

```javascript
// 消费记录
{
  id: string (UUID),
  amount: number (正数，单位为元),
  category: string ('food' | 'transport' | 'shopping' | 'entertainment' | 'living' | 'communication' | 'medical' | 'other'),
  note: string (可选),
  createdAt: ISO 时间戳,
  updatedAt: ISO 时间戳
}

// 应用状态
{
  transactions: Transaction[],
  settings: {
    currency: '¥'
  }
}
```

### localStorage Key
- `daily-expense-records`：消费记录 JSON 数组
- `daily-expense-settings`：设置对象

### 文件结构
```
/index.html    — 单页应用
/styles.css    — 全部样式
/app.js        — 应用逻辑
```

### 性能目标
- 首次渲染：< 500ms
- 记录保存：< 100ms
- 列表滚动：60fps
