## 一、产品概述

### 1.1 产品背景

现代人生活节奏快，消费场景多样（外卖、网购、交通、娱乐等），往往月底回顾时才发现"钱花哪了"。大多数人缺乏系统记账习惯，仅凭记忆难以准确分析消费结构。

### 1.2 产品定位

一款**轻量、快捷、无负担**的个人日常记账工具，专注于"记"这个核心动作，让用户用最少步骤完成每一笔消费记录。

### 1.3 目标用户

- 有记账意愿但缺乏坚持动力的职场人士
- 希望清晰了解每月消费结构的年轻人
- 追求简单工具、不需要复杂财务分析的用户

### 1.4 产品slogan

> **"一秒记账，清楚花钱"**

## 二、用户需求分析

### 2.1 核心痛点

| 痛点 | 描述 |
|------|------|
| 步骤太繁琐 | 现有记账App功能复杂，记账需要选择账户、分类、备注等大量操作 |
| 容易忘记 | 一天结束才想起来今天没记账，很多消费已经遗忘 |
| 难以坚持 | 三分钟热度，一周后就放弃使用 |
| 数据看不懂 | 记了一堆数字，但没有清晰的报表和分析 |
| 数据不安全 | 本地存储的数据容易因设备更换或重置而丢失 |

### 2.2 用户场景

| 场景 | 用户行为 |
|------|----------|
| 早餐买包子 | 扫码支付后，顺手上划快捷记账，3秒完成 |
| 淘宝购物 | 收到快递后，回顾订单快速补记 |
| 月底复盘 | 查看月度消费曲线，了解钱花在哪了 |
| 新设备使用 | 换手机后登录账号，同步历史记账数据 |

## 三、产品功能规划

### 3.1 功能优先级矩阵

| 功能 | 优先级 | 类型 |
|------|--------|------|
| 快速记账（首页一镜直达） | P0 | 核心功能 |
| 支出分类管理 | P0 | 核心功能 |
| 消费记录列表 | P0 | 核心功能 |
| **用户注册/登录** | P0 | 核心功能 |
| 本月消费概览 | P1 | 重要功能 |
| 月度消费统计 | P1 | 重要功能 |
| 数据导出 | P2 | 增值功能 |

### 3.2 功能详细说明

#### 3.2.1 P0 核心功能

**功能1：快速记账首页**

- **位置**：打开App第一屏
- **核心元素**：
  - 金额输入框（大字号，居中）
  - 分类快捷选择（横向滚动，常用分类置顶）
  - 备注快捷输入（可选）
  - 记账按钮
- **交互设计**：
  - 打开App即聚焦金额输入框，调起数字键盘
  - 数字输入完成即显示分类，直接点选
  - 记账完成后显示1秒完成动画，返回可继续记下一笔

**功能2：支出分类管理**

- **预设分类**（8个）：
  | 分类 | 图标 | 说明 |
  |------|------|------|
  | 餐饮 | 🍜 | 早中晚餐、外卖、零食 |
  | 交通 | 🚗 | 公交、地铁、打车、油费 |
  | 购物 | 🛒 | 日常用品、网购 |
  | 娱乐 | 🎮 | 游戏、电影、聚会 |
  | 居住 | 🏠 | 房租、水电煤 |
  | 通讯 | 📱 | 话费、流量 |
  | 医疗 | 💊 | 药店、医院 |
  | 其他 | 📦 | 无法归类的消费 |

- **分类支持自定义**（V2迭代）

**功能3：消费记录列表**

- **展示内容**：日期、分类图标、分类名称、金额、备注（如有）
- **交互能力**：
  - 点击单项可编辑或删除
  - 支持按日/周筛选
  - 默认按时间倒序排列

**功能4：用户注册/登录**

- **注册功能**：
  - 输入项：用户名（唯一标识）、密码（加密存储）
  - 用户名：6-20位字母或数字组合，不能为空
  - 密码：6-16位，不能为空
  - 注册成功后自动登录，并跳转至首页

- **登录功能**：
  - 输入项：用户名、密码
  - 支持记住登录状态（7天免登录）
  - 密码错误提示"用户名或密码错误"
  - 用户名不存在提示"该用户不存在"

- **会话管理**：
  - 使用Token机制进行身份认证
  - Token有效期：7天（记住登录）/ 24小时（普通登录）
  - Token存储在本地安全区域

- **安全考虑**：
  - 密码使用哈希存储（不可逆加密）
  - 登录失败锁定：连续5次失败后锁定15分钟
  - 不明文传输用户敏感信息

---

#### 3.2.2 P1 重要功能

**功能5：本月消费概览**

- **位置**：首页顶部或Tab切换
- **展示内容**：
  - 本月累计消费金额（大字号突出）
  - 与上月同期对比（涨/跌百分比）
  - 各分类消费占比（横向条形图）

**功能6：月度消费统计**

- **入口**：底部Tab"统计"
- **展示内容**：
  - 日历热力图：颜色深浅表示每日消费高低
  - 分类饼图：各分类金额占比
  - 月度消费折线图：近6个月趋势

---

#### 3.2.3 P2 增值功能

**功能7：数据导出**

- 导出格式：Excel（.xlsx）
- 导出内容：日期、分类、金额、备注
- 适用场景：做财务规划、报税参考

## 四、视觉设计规范

### 4.1 设计风格定位

- **风格关键词**：粉嫩、可爱、温暖、轻松
- **设计理念**：让记账成为一种愉悦的体验，而非负担。整体视觉传达"甜蜜生活"的感觉。
- **圆角风格**：大圆角设计（按钮、卡片圆角 16-24px），营造柔和亲切感

### 4.2 色彩系统

## 六、非功能性需求

### 6.1 性能要求

- App冷启动时间 < 2秒
- 记账操作完成到记录显示 < 500ms
- 列表滑动流畅，无卡顿（60fps）

### 6.2 隐私与安全

- 用户密码哈希存储，不可逆加密
- Token认证机制，保障会话安全
- 登录失败锁定机制，防止暴力破解
- 支持本地数据加密存储

### 6.3 兼容性

- 适配 iOS 14+ / Android 8.0+
- 主流全面屏适配

## 九、附录

### 9.1 竞品分析参考

| 竞品 | 优点 | 缺点 |
|------|------|------|
| 支付宝记账本 | 免安装、数据自动导入 | 依赖支付宝、分类粗 |
| 鲨鱼记账 | 功能完善、社区活跃 | 步骤多、广告多 |
| 随手记 | 专业、财务功能强 | 门槛高、偏重 |

### 9.2 名词解释

| 术语 | 说明 |
|------|------|
| P0 | 必须做，不做不成产品 |
| P1 | 很重要，会影响用户体验 |
| P2 | 锦上添花，可放到后续版本 |
| Token | 用户身份认证凭证，用于接口鉴权 |
| 哈希加密 | 不可逆的密码加密方式，保障密码安全 |
