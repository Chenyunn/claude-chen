## 一、产品介绍

### 1.1 什么是「每日记账」

每日记账是一款面向个人和家庭用户的日常收支管理工具。用户可以通过手机APP、电脑浏览器或微信小程序随时记录每一笔消费和收入，系统会自动汇总、分类、生成统计图表，帮助用户清晰了解自己的资金流向。

产品定位为**轻量级个人财务管理工具**，区别于复杂的企业财务软件，每日记账专注于三个核心需求：

- **记得快**：打开即记，三步完成一笔账
- **看得清**：自动生成月度汇总、分类占比、消费趋势
- **用得久**：数据云端同步，换手机不丢数据

### 1.2 产品形态

每日记账提供三种使用方式，数据互通：

| 使用方式 | 适用场景 | 运行平台 |
|---------|---------|---------|
| **Web网页版** | 办公室、家中电脑记账 | Chrome / Edge / Firefox / Safari |
| **移动APP** | 日常外出手机记账 | iOS / Android |
| **小程序** | 不想下载APP的轻量使用 | 微信小程序 |

同时，系统为运营团队提供**后台管理系统**，用于用户服务、数据监控和日常运维。

## 二、用户角色

每日记账系统涉及三类用户角色：

### 2.1 普通用户（记账用户）

使用每日记账进行个人收支管理的最终用户。可以：
- 注册账号并登录
- 记录每笔支出或收入
- 查看交易历史、搜索和筛选
- 查看消费统计和趋势分析
- 管理自己的分类和支付账户
- 编辑个人资料

### 2.2 运营管理员（Admin）

负责用户服务和日常运营的后台管理人员。可以：
- 查看平台数据总览（用户数、交易量等）
- 管理用户（查看详情、封禁/解封、重置密码）
- 查看全平台交易记录
- 管理系统分类
- 查看统计报表
- 处理导出任务

### 2.3 超级管理员（Super Admin）

拥有系统最高权限，除运营管理员的全部权限外，还可以：
- 创建和管理其他管理员账号
- 查看操作审计日志

## 三、功能模块说明

### 3.1 用户认证模块

**功能概述**：提供完整的用户身份管理，包括注册、登录、登出和会话维护。

**核心特性**：
- 用户名+密码注册，用户名限6-20位字母数字组合
- 密码加密存储（bcrypt），不保存明文
- 登录支持"7天免登录"（Remember Me）
- 连续5次密码错误自动锁定账号15分钟，防止暴力破解
- 登出后Token即时失效
- 登录态过期自动跳转登录页

**适用端**：Web端 ✅ | APP端 ✅

---

### 3.2 快速记账模块

**功能概述**：用户快速记录一笔支出或收入，是最核心的日常操作。

- 从预置分类中选择（餐饮、交通、购物、娱乐等）
- 选择支付账户（微信、支付宝、银行卡等）
- 选择交易日期（默认当天）
- 填写金额和备注（选填）
- 提交后立即刷新最近交易列表

**适用端**：APP端 ✅（完整可用） | Web端 ⚠️（界面完成，API对接中）

## 四、用户操作指南

### 4.1 注册账号

**Web端操作步骤**：

1. 打开浏览器，访问每日记账网站
2. 点击"还没有账号？立即注册"
3. 输入用户名（6-20位字母或数字）
4. 输入密码（6-20位），再次输入确认密码
5. 勾选"我已阅读并同意《服务协议》和《隐私政策》"
6. 点击"注册"按钮
7. 注册成功后自动跳转至首页

**APP端操作步骤**：

1. 打开每日记账APP
2. 在登录页点击"注册新账号"
3. 按提示填写用户名和密码
4. 勾选服务协议
5. 点击注册按钮
6. 成功后进入首页

### 4.2 登录

1. 进入登录页面
2. 输入已注册的用户名和密码
3. 如需7天内免登录，勾选"7天内免登录"
4. 点击"登录"按钮
5. 登录成功后进入首页

### 4.3 记一笔账（APP端）

1. 打开APP，默认进入首页（记账页）
2. 选择类型：支出 或 收入
3. 点击金额输入框，输入金额
4. 从分类网格中选择对应的消费类别（如"餐饮"）
5. 选择支付账户（如"微信支付"）
6. 确认日期（默认当天，可修改）
7. 可选：输入备注信息
8. 点击"保存账目"按钮
9. 提交成功后，下方"最近活动"区域立即更新

### 4.4 查看交易记录（APP端）

1. 点击底部导航栏"交易"标签
2. 进入交易记录页，显示按月汇总和交易列表
3. 可使用顶部搜索框输入关键词搜索
4. 点击分类标签（全部/餐饮/购物等）进行筛选
5. 向下滑动自动加载更多记录

### 4.5 查看统计分析（APP端）

1. 点击底部导航栏"统计"标签
2. 进入统计分析页，默认显示当月数据
3. 点击"按月/按季/按年"切换查看周期
4. 查看消费趋势折线图和分类占比饼图
5. 向下滑动查看支出排行明细

### 4.6 管理分类（Web端/APP端通用）

**添加分类**：
1. 进入"我的"页面
2. 找到"分类管理"区域
3. 选择"支出"或"收入"标签页
4. 点击"+"添加按钮
5. 在弹出的窗口中：
   - 输入分类名称（如"奶茶"）
   - 输入分类编码（英文，如 `milk_tea`）
   - 从emoji列表中选择图标
6. 点击"添加"确认

**编辑分类**：
1. 在分类列表中点击要修改的分类旁的编辑图标（✏️）
2. 修改名称或图标
3. 点击"保存"

**删除分类**：
1. 在分类列表中点击要删除的分类旁的删除图标（🗑️）
2. 确认删除
3. 注意：系统预置分类不可删除

### 5.2 前端技术

| 端 | 技术栈 | 说明 |
|---|--------|------|
| **Web端** | 原生 HTML5 + CSS3 + JavaScript (ES2017) | 无框架依赖，纯静态页面，零构建 |
| **移动APP** | uni-app 3.x + Vue 3 + TypeScript + Vite 5 | 一套代码编译到iOS、Android、H5和11种小程序 |
| **后台管理** | 待定 | PRD已完成，技术选型待定 |

### 5.3 后端技术

| 服务 | 技术栈 | 说明 |
|------|--------|------|
| **认证服务** | Node.js + Express 4 + MySQL2 | 处理注册/登录/鉴权，JWT Token管理 |
| **业务服务** | Java Spring Boot + MyBatis | 处理交易CRUD、统计查询、账户管理 |
| **数据库** | MySQL 9.x | 9张业务表 + 系统分类数据 |

### 5.4 数据安全

- 用户密码使用 bcrypt（12轮哈希）加密存储
- 接口认证采用 JWT Bearer Token 机制
- 敏感操作（登录失败计数、账号锁定）有防暴力破解保护
- 数据软删除设计，避免误操作导致数据永久丢失
- API接口采用参数化查询，防止SQL注入

### 5.5 跨端能力

每日记账APP基于uni-app框架构建，一套代码支持以下平台：

| 平台类型 | 具体平台 | 状态 |

## 七、常见问题

### 账号相关

**Q：忘记密码怎么办？**

当前版本暂不支持自助找回密码。如忘记密码，请联系客服协助重置。后续版本将增加手机号/邮箱验证找回功能。

**Q：可以注销账号吗？**

当前版本暂无自助注销功能。如需注销，请联系客服处理。

**Q：为什么提示"账号已被锁定"？**

连续输入错误密码5次，账号会自动锁定15分钟。这是系统安全保护机制。请等待15分钟后再尝试，或联系客服。

**Q：为什么退出登录后，用原来的链接还能看到数据？**

这是v1.0版本已知的安全问题，退出后Token未完全失效。我们正在修复中，预计下一版本解决。当前请确保在不使用的公共设备上手动清除浏览器缓存。

### 使用相关

**Q：记账数据保存在哪里？**

所有记账数据保存在云端服务器数据库中。只要使用同一账号登录，换手机、换电脑都能看到自己的数据。

**Q：系统分类和自定义分类有什么区别？**

系统分类是预置的10个常用分类（餐饮、交通等），所有用户共享，不可删除。自定义分类是用户自己创建的，可以随时增删改。

**Q：可以导出数据吗？**

当前版本的数据导出功能尚在开发中。后续版本将支持导出为Excel/CSV格式。当前如需数据备份，请联系客服。

**Q：删除的分类还能恢复吗？**

分类采用软删除设计，数据标记为删除但保留在数据库中。普通用户无法恢复，如需恢复，请联系客服协助。

### 跨端相关

**Q：Web端和APP端数据互通吗？**

是的。无论通过Web端还是APP端记账，数据都存储在同一个云端数据库中。登录同一账号即可看到全部数据。

**Q：为什么小程序有些功能用不了？**

当前小程序（微信等）需要在对应平台填写appid并提交审核后才能正常使用。开发阶段仅可在开发者工具中本地预览。

**Q：APP需要什么手机系统版本？**

iOS建议12.0及以上，Android建议8.0及以上。

### 安全相关

**Q：我的数据安全吗？**

我们采取多层安全措施保护您的数据：密码加密存储、传输过程加密、数据库参数化查询防止注入攻击。但请注意，当前开发版本不应存储真实敏感数据。

**Q：为什么说当前版本不建议生产环境使用？**

v1.0版本存在以下待修复问题：
- 退出登录后Token未完全失效
- 部分URL使用HTTP明文传输
- 前端存在XSS注入风险

这些问题将在正式发布前修复完毕。
