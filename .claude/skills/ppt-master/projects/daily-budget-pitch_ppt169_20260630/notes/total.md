# 01_cover

各位好，我是「每日记账」的创始人。先用一句话说清我们做的事：让记账，像发一条朋友圈一样轻。

记账这件事，几乎人人都试过，但绝大多数人都坚持不下来。我们要做的，就是把「记一笔」这个动作做到极致简单——五秒内完成——再用习惯养成机制，解决记账类产品最大的难题：留不住人。

产品已经覆盖 Web、iOS、安卓和微信小程序三端。今天来，是寻求天使轮融资。接下来五页，我会讲清楚痛点、卖点、产品进展、市场和我们的诉求。

# 02_problem

先说痛点。这是一个很拧巴的市场：人人都想记账，但没人能坚持。

我们调研下来，放弃记账的原因高度集中在四点：第一，步骤太繁琐，选账户、选分类、填备注，记一笔要点七八下；第二，容易忘，一天结束才想起没记；第三，三分钟热度，下载一堆 App 一周后全删了；第四，记了一堆数字，却看不懂、没法复盘。

为什么现有产品解决不了？因为它们走了两个极端：随手记、鲨鱼记账这类太重，功能堆得满、广告还多；支付宝记账本又绑生态、分类粗。真正的市场缺口，是一款够轻、无广告、还能让人坚持下来的工具——这就是我们切进去的位置。

# 03_value

这一页是重点：我们凭什么赢？三个字——轻、快、黏。

先说快，这是我们的核心护城河。一镜直达的首页，金额、分类、保存三步搞定，打开就聚焦输入框，目标是单笔五秒内完成，比任何竞品都快。

再说轻：无广告、不绑生态、零学习成本，加上粉嫩温暖的产品调性，让记账从一种负担变成愉悦的小习惯。

最关键的是黏——连续打卡、每周账单、餐饮专项复盘，用习惯养成机制直接打到行业最痛的留存难题。

下面这排数字是我们的产品目标：单笔小于五秒、次日留存超过百分之四十、人均每天记账两笔以上、零广告零干扰。这些不是空话，是我们产品设计里就埋好的指标。

# 04_product

讲讲产品进展，证明我们不是只有 PPT。三端已经落地，核心链路是真实跑通的。

已经实现的部分：注册登录，用的是 JWT 加 bcrypt 加密；快速记账和分类管理，十个预置分类加自定义；交易记录支持按日分组、搜索、筛选、分页;还有餐饮专项分析和统计图表。

技术上，后端是 Spring Boot 加 MyBatis 加 MySQL，移动端用 uni-app 一套代码发三端，研发效率高。

现在的阶段是：MVP 已经成型，核心链路打通，正在进入内测打磨期。

这里我要特别诚实地说一点：云同步、账单导出、后台运营系统，这些是下一阶段的功能，目前还没上线。我在这一页如实标注了，不会给各位画超出当前能力的饼。

# 05_market_ask

最后，市场、路线图和我们的诉求。

市场上，移动支付的年轻用户基数在 2.6 亿量级——这是行业测算口径；记账是每天两次以上的高频刚需，天然有粘性；而我们的差异化很清楚：做最轻、无广告的那一个，避开和巨头的正面战场。

路线图分三步：零到三个月是种子期，核心打磨加五十到两百人内测，跑通激活；三到九个月是增长期，上线云同步、铺内容矩阵，把七日留存做到百分之二十以上；九个月以后进入商业化，探索账单导出、会员和数据洞察。

我们这轮寻求天使轮融资，主要用于团队搭建、产品打磨和冷启动获客；十二个月的里程碑，是做到十万注册用户，验证留存和单位经济模型。

让记账重新变得简单——一秒记账，清楚花钱。期待与各位同行，谢谢。
