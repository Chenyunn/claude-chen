# 周报生成 (Weekly Report)

根据 git 提交记录自动汇总本周工作内容，整理成结构化的中文周报，并保存为 Markdown 文档。

## 工作流程

### 1. 确定本周时间范围

以「今天」为基准，计算本周的起止日期（周一 00:00 到当前时刻）。也可根据用户指定的范围调整（如「上周」「最近两周」）。

### 2. 读取提交记录

优先读取本地 git 历史。先确保拿到最新数据：

```bash
git fetch --all --quiet 2>/dev/null || true
```

获取本周提交（按作者过滤可选，默认取当前用户）：

```bash
git log --since="2026-06-22 00:00:00" --until="2026-06-27 23:59:59" \
  --pretty=format:"%h|%an|%ad|%s" --date=format:"%Y-%m-%d %H:%M" --no-merges
```

说明：
- `--no-merges` 过滤掉合并提交，让周报聚焦实际工作。
- 如需只统计自己的提交，追加 `--author="$(git config user.name)"`。
- 需要查看每次提交改动的文件时，对关键提交补充执行 `git show --stat <hash>`。
- 若本地历史不全，可通过 `gh`（GitHub CLI）补充：`gh api repos/{owner}/{repo}/commits --jq ...`；仓库地址见 `git remote -v`。当前远程为 `https://github.com/Chenyunn/claude-chen.git`。

### 3. 汇总与归类

将提交按主题归类（如：功能开发、Bug 修复、文档、重构、运维等），合并同类项，用业务语言而非纯技术语言描述。识别 commit message 中的前缀（feat/fix/docs/refactor/chore 等）辅助归类。

避免逐条罗列 commit；要提炼成「做了什么、解决了什么问题、产出了什么」。

### 4. 生成周报文档

按下方模板生成，保存到 `000.文档纪要/` 目录（若不存在则创建）。文件命名：`周报-YYYY年第WW周-YYYYMMDD至YYYYMMDD.md`，例如 `周报-2026年第26周-20260622至20260628.md`。

## 周报模板

```markdown
# 工作周报

- **周期**：YYYY-MM-DD 至 YYYY-MM-DD（第 WW 周）
- **汇报人**：<git user.name>
- **生成日期**：YYYY-MM-DD

## 一、本周工作概述

<2-3 句话总结本周整体进展与重点。>

## 二、本周完成工作

### 1. 功能开发
- <按业务归纳的事项，附关键提交 hash>

### 2. 问题修复
- <...>

### 3. 文档与其他
- <...>

## 三、数据统计

- 本周提交数：<N> 次
- 涉及模块/目录：<列表>

## 四、下周计划

- <根据未完成项或 TODO 推断，若无则留待补充>

## 五、附录：提交明细

| 日期 | 提交 | 说明 |
|------|------|------|
| YYYY-MM-DD | `hash` | commit message |
```

## 注意事项

- 全程使用中文输出。
- 「下周计划」如无明确依据，可基于进行中的工作合理推断，并标注「待确认」。
- 保存前确认 `000.文档纪要/` 目录存在，使用文件写入工具创建文档（不要用 echo 重定向）。
- 生成后向用户报告文档保存路径。
